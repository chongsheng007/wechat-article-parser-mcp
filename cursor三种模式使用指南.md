# Cursor 三种 AI 模式使用指南

## 📋 概述

Cursor 左下角的眼睛图标提供了三种 AI 交互模式，每种模式有不同的特点和使用场景。

---

## 🎭 三种模式对比

### 角色定位（开发应用类比）

| 模式 | 角色 | 职责 | 是否执行操作 | 输出 |
|------|------|------|------------|------|
| **Plan** | 架构师 | 设计、规划、制定方案 | ❌ 不执行 | 计划、架构、步骤 |
| **Agent** | 开发者 | 实现、执行、完成任务 | ✅ 执行 | 代码、文件、成果 |
| **Ask** | 评估师 | 审查、评估、提供建议 | ❌ 不执行 | 建议、问题、最佳实践 |

---

## 🔍 详细说明

### 1. Plan（规划模式）- 架构师

#### 特点
- ✅ 制定计划，不立即执行
- ✅ 分析任务，给出详细步骤
- ✅ 不会修改代码或文件
- ✅ 适合需要先看方案再决定的情况

#### 使用场景
- 规划项目结构
- 设计系统架构
- 制定开发计划
- 分析任务步骤
- 论文大纲规划

#### 示例
```
输入："帮我规划如何重构这个项目"

Plan 会：
1. 分析项目结构
2. 列出需要重构的地方
3. 给出详细的执行步骤
4. 但不会立即执行

你可以：
- 查看计划
- 决定是否执行
- 让 Agent 模式来执行
```

---

### 2. Agent（执行模式）- 开发者

#### 特点
- ✅ 可以执行任务
- ✅ 可以修改代码
- ✅ 可以创建文件
- ✅ 更自主，会主动执行操作
- ✅ **这是你当前使用的模式**

#### 使用场景
- 创建新文件
- 修复 bug
- 重构代码
- 添加新功能
- 写论文
- 解析文章

#### 示例
```
输入："帮我创建一个新文件"

Agent 会：
- 直接创建文件
- 写入内容
- 完成操作

输入："解析这篇微信公众号文章"

Agent 会：
- 调用 MCP 工具
- 解析文章
- 返回结果
```

---

### 3. Ask（问答模式）- 评估师

#### 特点
- ✅ 只回答问题，不执行操作
- ✅ 不会修改代码
- ✅ 不会创建文件
- ✅ 适合咨询、解释、建议

#### 使用场景
- 代码解释
- 问题诊断
- 最佳实践咨询
- 代码审查
- 学术规范咨询

#### 示例
```
输入："这段代码有什么问题？"

Ask 会：
- 分析代码
- 指出问题
- 给出建议
- 但不会修改代码

如果想修改，需要切换到 Agent 模式
```

## 🎯 实际应用场景

### 场景 1：开发应用

#### 完整流程

```mermaid
graph LR
    A[Plan: 设计方案] --> B[Ask: 评估方案]
    B --> C[Agent: 实现功能]
    C --> D[Ask: 代码审查]
    D --> E[Agent: 优化代码]
```

#### 步骤详解

1. **Plan（架构师）**：设计方案
   ```
   输入："帮我设计一个用户登录系统"
   
   输出：
   - 需要哪些模块
   - 数据库设计
   - API 接口设计
   - 安全考虑
   - 技术选型
   ```

2. **Ask（评估师）**：评估方案
   ```
   输入："这个架构设计有什么问题？"
   
   输出：
   - 潜在问题
   - 改进建议
   - 最佳实践
   ```

3. **Agent（开发者）**：实现功能
   ```
   输入："按照这个架构开始实现"
   
   输出：
   - 实际代码
   - 创建文件
   - 实现功能
   ```

4. **Ask（评估师）**：代码审查
   ```
   输入："这段代码有什么问题？"
   
   输出：
   - 代码审查
   - 优化建议
   ```

5. **Agent（开发者）**：优化代码
   ```
   输入："修复这些问题"
   
   输出：
   - 优化后的代码
   ```

---

### 场景 2：写论文

#### 完整流程

1. **Plan（架构师）**：规划结构
   ```
   输入："帮我规划一篇关于微信公众号解析技术的论文结构"
   
   输出：
   - 论文大纲
   - 各章节安排
   - 写作顺序
   - 参考文献组织
   ```

2. **Ask（评估师）**：评估大纲
   ```
   输入："这个大纲是否符合学术规范？"
   
   输出：
   - 结构评估
   - 改进建议
   ```

3. **Agent（开发者）**：开始写作
   ```
   输入："按照大纲开始写论文"
   
   输出：
   - 实际论文内容
   - 各章节初稿
   ```

4. **Ask（评估师）**：审查内容
   ```
   输入："这段文字是否学术严谨？"
   
   输出：
   - 文字审查
   - 改进建议
   ```

5. **Agent（开发者）**：修改完善
   ```
   输入："改进这些地方"
   
   输出：
   - 修改后的内容
   ```

---

### 场景 3：阅读文章

#### 完整流程

1. **Agent（开发者）**：解析文章
   ```
   输入："解析这篇微信公众号文章：https://..."
   
   输出：
   - 文章内容
   - 元数据
   - 图片
   ```

2. **Ask（评估师）**：分析内容
   ```
   输入："总结这篇文章的核心观点"
   
   输出：
   - 内容分析
   - 核心观点
   ```

3. **Plan（架构师）**：规划整理
   ```
   输入："规划如何将这些文章整理成文献综述"
   
   输出：
   - 整理方案
   - 分类方法
   - 综述结构
   ```

## 💡 最佳实践

### 推荐工作流程

1. **先 Plan（规划）** - 明确目标和步骤
2. **再 Ask（评估）** - 审查方案的可行性
3. **最后 Agent（执行）** - 实际完成任务
4. **循环 Ask（评估）** - 持续改进

### 使用原则

- **Plan**：用于复杂任务，需要先规划
- **Agent**：用于需要实际操作的场景（**最常用**）
- **Ask**：用于咨询、审查、评估

---

## 🎓 针对写论文和阅读文章的建议

### 写论文推荐流程

1. **Plan → Ask → Agent → Ask → Agent**

   ```
   1. Plan: 规划论文结构
   2. Ask: 评估大纲合理性
   3. Agent: 开始写作
   4. Ask: 审查内容质量
   5. Agent: 修改完善
   ```

### 阅读文章推荐流程

1. **Agent → Ask → Plan**

   ```
   1. Agent: 解析文章（使用 MCP 工具）
   2. Ask: 分析内容
   3. Plan: 规划如何整理
   ```

## 📝 快速参考

### 什么时候用 Plan？

- ✅ 需要规划复杂任务
- ✅ 需要设计方案
- ✅ 想先看计划再决定
- ✅ 需要分析任务步骤

### 什么时候用 Agent？

- ✅ 需要实际操作（**最常用**）
- ✅ 需要修改代码
- ✅ 需要创建文件
- ✅ 需要解析文章
- ✅ 需要写论文

### 什么时候用 Ask？

- ✅ 需要咨询问题
- ✅ 需要代码审查
- ✅ 需要评估质量
- ✅ 需要最佳实践建议

---

## 🔧 切换模式

在 Cursor 左下角的眼睛图标中：

1. 点击 **"Plan"** → 规划模式
2. 点击 **"Agent"** → 执行模式（**当前使用**）
3. 点击 **"Ask"** → 问答模式

---

## 📌 总结

- **Plan（架构师）**：设计、规划，不执行
- **Agent（开发者）**：实现、执行，实际操作
- **Ask（评估师）**：审查、评估，提供建议

最佳实践：**Plan → Ask → Agent → Ask → Agent** 的循环流程
